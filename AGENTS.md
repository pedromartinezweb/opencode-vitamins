# Project Instructions

Act professionally. Use SOLID principles, clean architecture, and clear responsibility boundaries.

Write simple code with short, descriptive names. Avoid comments unless a technical decision is not clear from the code.

Before coding, understand the goal, split the work into verifiable tasks, and define acceptance criteria.

After coding, run available tests and state what was verified. If tests do not exist, add the smallest useful tests for the new behavior.

Do not mix unrelated changes. Do not perform broad refactors unless required to complete the task safely.
