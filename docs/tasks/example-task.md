# Task: Example Product POC

Build a small full-stack product POC.

## Goal

Create a working application with:

- Frontend.
- Backend.
- Database.
- Local run instructions.
- Automated checks where practical.

## Product Context

The product should help a small team manage a simple operational workflow. Choose a pragmatic domain model, keep the scope small, and make the first screen useful.

## Requirements

- Create a clean, responsive UI.
- Expose backend APIs for the main entities.
- Store data in a local database.
- Include realistic seed data.
- Keep code organized by responsibility.
- Add tests for core business logic.
- Add README commands to run and verify the project.

## Acceptance Criteria

- The app starts locally with documented commands.
- The UI shows seeded data.
- Users can create, update, and list the main entities.
- The backend exposes a health endpoint.
- Core business logic has automated tests.
- The final response includes commands run and verification result.
